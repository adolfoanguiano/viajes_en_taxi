# Análisis para una compañia de viajes en taxi de aplicación 🚕🚕🚕

**Zuber** es una compañia de viajes compartidos en taxi en la ciudad de Chicago, Estados Unidos.

Por petición de la empresa, con los datos recabados en sus viajes, se solicitó realizar un análisis con la finalidad de encontrar patrones de los viajes, usuarios, permitiendo comprender las preferencias de los usuarios y si hay factores externos que impacten en los viajes.

Al final se planteo la hipótesis para saber en específico si "La duración promedio de los viajes desde el Loop hasta el Aeropuerto Internacional O'Hare cambia los sábados lluviosos".

## Desarrollo del proyecto

* Se proporcionaron 2 bases de datos, una con los viajes realizados por cada una de las diferentes compañías de servicio de taxi; y la otra base de datos con información del promedio de viajes realizados a los principales destinos (por zona) de Chicago.
* Se cargaron , se analizaron y se prepararon cada una de las bases de datos para poder empezar a realizar los análisis, lo que ayudó a identificar:
    **a)** las principales 10 zonas de destino de los viajes.
    **b)** se realizó un gráfico para mostrar el número de viajes de cada compañia de taxis de dos días.
    **c)** se realizó una gráfica para mostrar el promedio de viajes por cada una de las zons destino.
  
* Se planteo la hipótesis y se utilizó un dataset que contiene los datos sobre los viajes desde el barrio de Loop hasta el Aeropuerto Internacional O'Hare y las condiciones climatológicas.
* Se realizó una prueba estadísitica para probar la hipótesis, resultando que un factor externo, como es la lluvia, afecta en la cantidad y en la duración de los viajes.
